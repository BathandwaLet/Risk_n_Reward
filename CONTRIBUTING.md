Contributing Guide
How to Contribute
I welcome contributions! Here's how:

1. Find Something to Work On
Check the Issues tab for bugs or feature requests

Look for issues tagged good first issue if you're new

Or suggest your own improvement(s)

2. Make Your Changes
Fork the repository

Create a branch for your work:

bash
git checkout -b fix/description
or
git checkout -b feature/new-game
Make your changes

Test them:

bash
dotnet build
dotnet test   (when available)
3. Code Standards
Use clear variable names

Add comments for complex logic

Keep it simple - this is an educational project

No real money/gambling code

4. Submit a Pull Request
Push your branch: git push origin your-branch-name

Open a Pull Request on GitHub

Describe what you changed and why

Link any related issues

What We're Looking For
Bug fixes - something is not working right

Small features - simple improvements

Documentation - clearer explanations

Code cleanup - better organisation

Questions?
Open a Discussion on GitHub

Comment on the issue you're working on

Keep conversations friendly and helpful
